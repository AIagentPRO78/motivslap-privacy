# redstack M2 lab

Deliberately vulnerable, MacBook-local environment for exercising the
full redstack engagement flow end-to-end. Everything binds to
`127.0.0.1` only. **Never expose this lab to the internet** — every
component ships with weak or default credentials and publicly readable
fake secrets by design.

## What's in it

| Component        | Port         | Purpose                                                           |
| ---------------- | ------------ | ----------------------------------------------------------------- |
| DVWA             | 8080         | Classic deliberately-vulnerable web app for `/web-app`.           |
| minio (S3-compat)| 9000, 9001   | Object store with a public bucket and a fake `terraform.tfstate`. |
| Postgres         | 5432         | Seeded with a synthetic `users` table (clearly fake identifiers). |
| Keycloak         | 8081         | Demo realm with deliberate OIDC + password-policy misconfigs.     |
| k3d cluster      | 6443         | `redstack-lab` cluster with vulnerable manifests for `/cloud-audit`. |

Fake credentials use the prefix `FAKEKEYLAB` so static scanners match
them as test fixtures without ever being valid for any real service.

## Requirements

- macOS 12+ (Apple Silicon or Intel).
- OrbStack **or** Docker Desktop running.
- `k3d` installed (`brew install k3d`).
- `kubectl` installed (`brew install kubectl`).

## Bring it up

```bash
cd redstack/lab
./up.sh
```

Expected completion: ≤5 minutes on an M-series Mac (image pulls
dominate the first run; subsequent runs are ~30 seconds).

## Tear it down

```bash
./down.sh
```

Removes containers, volumes, the k3d cluster, and seeded data.

## Exercise from Claude Code

With the redstack SKILL.mds installed at `~/.claude/skills/redstack/`,
open Claude Code in a new terminal and run:

```
/office-hours     # intake "Example Corp" lab customer (scripted)
/engagement       # load lab/scope.example.yaml.signed (demo signature)
/recon            # should discover minio public bucket + k3d ingress
/web-app          # should find DVWA vulnerabilities
/cloud-audit      # should flag the privileged pod + cluster-admin SA
/triage           # dedupe + CVSS
/reporter         # draft deliverable.md
```

The demo signature on the example scope file is generated by
`./up.sh` with a lab-only key pair (persisted under `.lab-keys/` in
this directory, gitignored). The key never leaves the lab.

## Safety notes

- Binds to 127.0.0.1 only. If your docker daemon exposes ports on
  0.0.0.0, reconfigure before bringing up the lab.
- The public minio bucket is reachable from any process on your Mac.
  Remember to `./down.sh` when you finish.
- DVWA's default credentials are `admin / password`. Intentional.
- The k3d cluster has a ServiceAccount bound to `cluster-admin`.
  Intentional. Don't point it at anything outside the lab.
